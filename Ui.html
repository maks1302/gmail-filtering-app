⚠️ Common Pattern Mistakes
🧭 Exact vs Regex
Every condition starts in Exact mode. Turn on the Regex toggle only when you want pattern matching.
✓ Exact mode
example.us@gmail.com
From matches this exact sender
✓ Regex mode
.*@gmail\.com
From matches any Gmail sender
✓ Body regex
\blet me in\b
Body matches the whole phrase only
In Regex mode, . means any character and \. means a literal dot. In Exact mode, text is matched literally.
🐛 Body matching old quoted text
When you reply to an email, Gmail includes the original message in the body. A body rule for "let me in" will match any reply thread that previously contained those words, even from a different sender.
✓ Always combine body rules with a from condition using AND logic.
Pattern Guide
Mode Pattern Meaning Example
Exact hello Literal text match Good for simple subject/body phrases
Exact example.us@gmail.com Exact sender email Safe default for From / To
Regex \bhello\b Whole word only Won't match "hellofresh"
Regex sale|promo|deal OR — any of these Matches if any word found
Regex @domain\.com Literal domain in regex Use \. for literal dot
Regex .*@gmail\.com Any Gmail sender Good for From / To regex rules
Regex ^.*<ceo@company\.com>$ Exact formatted sender line Useful when Gmail shows Name <email>
Regex ^hello Starts with Field begins with "hello"
Regex hello$ Ends with Field ends with "hello"

Verify new patterns with Test or 🐛 Debug before enabling destructive actions.
